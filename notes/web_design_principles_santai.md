## 1. Proximity

### Principle

Group related elements together to show relationships.

### Example

Bad: - Label far from input field

Good: - "Email" label directly above input box

### Agent Hint

If elements are related → reduce spacing between them.

------------------------------------------------------------------------

## 2. Fitts's Law

### Principle

Bigger + closer targets are easier to interact with.

### Example

Bad: - Tiny "Submit" button in corner

Good: - Large centered CTA button

### Agent Hint

Primary actions → large + easy to reach.

------------------------------------------------------------------------

## 3. Emphasis

### Principle

Highlight the most important element.

### Example

Bad: - All text same size/color

Good: - Big bold headline + colored CTA

### Agent Hint

Ensure exactly 1 primary focal point per screen.

------------------------------------------------------------------------

## 4. White Space

### Principle

Empty space improves clarity and focus.

### Example

Bad: - Crowded UI with no spacing

Good: - Padding between sections and elements

### Agent Hint

If UI feels crowded → increase spacing.

------------------------------------------------------------------------

## 5. Color Palette

### Principle

Limit colors and use intentionally.

### Example

Bad: - 10 random colors

Good: - Neutral base + 1--2 accent colors

### Agent Hint

Use color primarily for emphasis, not decoration.

------------------------------------------------------------------------

## 6. Layout Structure

### Principle

Organize content into clear sections.

### Example

Bad: - Mixed content with no hierarchy

Good: - Sections: Hero → Features → CTA

### Agent Hint

Each section = one idea.

------------------------------------------------------------------------

## 7. Consistency

### Principle

Keep styles uniform across the product.

### Example

Bad: - Different button styles everywhere

Good: - Same button design system-wide

### Agent Hint

Reuse components instead of reinventing.

------------------------------------------------------------------------

## 8. Modularity

### Principle

Design reusable components.

### Example

Bad: - Hardcoded layouts everywhere

Good: - Reusable cards, sections, components

### Agent Hint

Abstract repeating UI into components.

------------------------------------------------------------------------

## 9. Anchor Objects

### Principle

Use strong visuals to guide attention.

### Example

Bad: - No clear visual hierarchy

Good: - Large image or headline at section start

### Agent Hint

Start sections with a visual anchor.

------------------------------------------------------------------------

## 10. Scanning Patterns

### Principle

Users scan in Z or F patterns.

### Example

Z-pattern: - Landing page → headline → CTA → image

F-pattern: - Blog → headings on left, content across

### Agent Hint

Place key info along natural scan paths.

------------------------------------------------------------------------

## Global Rules for Agents

1.  Prioritize clarity over creativity
2.  Always establish hierarchy
3.  Minimize cognitive load
4.  Optimize for scanning, not reading
5.  Default to simple, spaced layouts

------------------------------------------------------------------------

## Mental Model

Design = guiding user attention.

Goal: - Show what matters - Make it easy to act
